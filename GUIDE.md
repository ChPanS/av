# av · livecoding — справочник по визуальным uniforms

Как музыка из Strudel превращается в «ручки» (uniforms) для шейдера. Логика — в `bridge.js`,
объявление/прокидка в шейдер — авто-генерируются в `renderer.js` из объекта `uniforms`.

---

## 1. Поток данных

```
Strudel scheduler → вывод (audio.js) → handleHap (bridge.js) → объект uniforms
renderer.js: каждый кадр гасит огибающие (decay) и прокидывает uniforms в шейдер
GLSL: твоя функция mainImage(out vec4 fragColor, in vec2 fragCoord)
```

Каналы делятся на три типа: **импульсы** (ударные), **инструменты** (велосити + питч),
**глобальные** (часы/энергия/цвет).

---

## 2. Полный список uniforms

### Ударные — импульсы (всплеск + быстрый спад)
| Uniform | Спад | Канал |
|---|---|---|
| `uKick` | быстрый (0.86), до ~1.5 | бочка |
| `uSnare` | быстрый (0.85) | снейр |
| `uClap` | быстрый (0.85) | клэп (отдельно от снейра) |
| `uHat` | оч. быстрый (0.80) | закрытый хэт |
| `uOpenHat` | быстрый (0.82) | открытый хэт |

### Инструменты — у каждого ДВА юниформа
`u<Name>Vel` — велосити-огибающая, плавно спадает 1→0 (как громкость ноты), много шагов.
`u<Name>Pitch` — высота последней ноты этого инструмента (класс в октаве, 0..1), держится.

| Инструмент | Vel-спад | Юниформы |
|---|---|---|
| Pad | плавный (0.94) | `uPadVel` `uPadPitch` |
| Atmosphere | очень плавный (0.965) | `uAtmoVel` `uAtmoPitch` |
| Key | средний (0.90) | `uKeyVel` `uKeyPitch` |
| Lead | средний (0.91) | `uLeadVel` `uLeadPitch` |
| Bass | плавный (0.92) | `uBassVel` `uBassPitch` |
| Arp | быстроватый (0.88) | `uArpVel` `uArpPitch` |
| FX | плавный (0.93) | `uFxVel` `uFxPitch` |
| Vox | плавный (0.93) | `uVoxVel` `uVoxPitch` |

### Глобальные
| Uniform | Диапазон | Смысл |
|---|---|---|
| `uResolution` | px (vec2) | размер кадра |
| `uTime` | растёт | секунды с запуска |
| `uPitch` | 0..1 | высота последней ноты вообще (класс в октаве) |
| `uHue` | 0..1 | оттенок, плавно ведётся за нотой |
| `uEnergy` | 0..1 | плотность/громкость потока (прокси интро↔дроп) |
| `uPad` | 0..~1.2 | СОВМЕСТИМОСТЬ: общая гармоническая огибающая (старые шейдеры) |
| `uBeat` | 0..N | позиция в лупе в циклах; `floor` = номер такта |
| `uLoop` | 0..1 | позиция в лупе нормированная |
| `uBeatFrac` | 0..1 | фаза внутри текущего цикла |

Заметки:
- **Велосити** инструмента — это плавная огибающая «нажали клавишу → звук гаснет»: при
  событии прыгает к ~громкости ноты и плавно спадает к нулю. Разные инструменты гаснут
  с разной скоростью (атмосфера тянется дольше всех, арп — короче).
- **Питч** инструмента держит последнее значение (не гаснет), как и глобальный `uPitch`.
- `uPad` оставлен для совместимости со старыми шейдерами — в него по-прежнему стекаются
  все гармонические инструменты.

## 3. Что первично: тег `.vis()` → иначе угадывание

```
1. есть hap.value.vis (явный тег)?  → используем ТОЛЬКО его
2. тега нет                         → угадываем по имени звука (грубо)
3. в любом случае: нота → uPitch/uHue (глобальные), и uEnergy += gain
```

Тег действует на всё событие. Тонкие каналы (`atmosphere`, `lead`, `arp`, `fx`, `vox`)
по имени **не угадываются** — их задаёт только тег. Автоугадывание даёт лишь грубое
деление: kick/snare/clap/hat/oh/bass/pad/key.

---

## 4. Теги `.vis("...")`

Вешается на КОНКРЕТНУЮ дорожку (не на весь `stack`):

```js
stack(
  s("myKick").vis("kick"),
  s("clap_loop").vis("clap"),
  note("c2 g1").s("sawtooth").vis("bass"),
  note("<...>").s("gm_epiano").vis("key"),
  note("c5 e5").s("triangle").vis("arp"),
  s("vocalchop").vis("vox")
)
```

| Тег | Канал(ы) |
|---|---|
| `kick` | `uKick` |
| `snare` | `uSnare` |
| `clap` | `uClap` |
| `hat` | `uHat` |
| `oh` / `openhat` | `uOpenHat` |
| `pad` | `uPadVel` + `uPadPitch` (+ `uPad`) |
| `atmosphere` / `atmo` | `uAtmoVel` + `uAtmoPitch` |
| `key` / `keys` / `chord` | `uKeyVel` + `uKeyPitch` (+ `uPad`) |
| `lead` / `melody` | `uLeadVel` + `uLeadPitch` |
| `bass` / `sub` | `uBassVel` + `uBassPitch` |
| `arp` | `uArpVel` + `uArpPitch` |
| `fx` | `uFxVel` + `uFxPitch` |
| `vox` / `vocal` | `uVoxVel` + `uVoxPitch` |

## 5. Угадывание по имени (когда тега нет)

| Условие на имя | Канал |
|---|---|
| `bd` | kick |
| `cp` / `clap` | clap |
| `sn` / `rim` | snare |
| `oh` / `open` | openhat |
| `hh` / `hat` | hat |
| `bass` / `sub` / `reese` / `808` | bass |
| `piano` / `epiano` / `rhodes` / `keys` | key |
| `pad` / `string` / `atmos` | pad |
| прочие синты (`saw` `tri` `squ` `sine` `super` `gm_`) | pad |
| иначе | в каналы не идёт (нота и энергия учитываются) |

---

## 6. Как добавить свой uniform

1. `bridge.js` — поле в `uniforms`; для гаснущего — строка в `decay`; правило в `route`/`hitInstrument`.
2. `renderer.js` — НИЧЕГО: объявление в шейдере и прокидка генерируются из `uniforms` автоматически.
3. (опц.) тег в `route`, если хочешь управлять им через `.vis()`.

---

## 7. Подводные камни

- **Тег на дорожку, не на `stack`.**
- **`u<Name>Pitch` — класс в октаве (0..1)**, без различения октав; для «высоты вообще»
  складывай с уровнем громкости/энергии в шейдере.
- **Аккорд = несколько событий разом** → велосити/энергия получают суммарный вклад.
- **`uKick` до ~1.5, `uPad` до ~1.2** — не нормированы в 0..1, учитывай в шейдере.
- **Подсветка нот в редакторе** опирается на позиции из транспайлера; работает для кода
  паттерна (не шейдера).
